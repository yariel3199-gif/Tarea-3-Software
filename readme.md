## 📌 Descripción General

Este repositorio contiene los entregables correspondientes a la fase de **Arquitectura y Diseño** dentro de un proyecto de software.

El objetivo es documentar la estructura del sistema, las decisiones arquitectónicas tomadas y el diseño técnico, así como proporcionar representaciones visuales mediante diagramas UML y plantillas reutilizables.

---

## 📦 Contenido del Proyecto

### 📄 Documento de Arquitectura y Diseño

Archivo: `Tarea3_Arquitectura_Diseno.pdf`

Incluye:

* Descripción de la arquitectura del sistema
* Registro de decisiones arquitectónicas (ADR)
* Diseño técnico del sistema
* Explicación de diagramas UML

---

### 🧠 Decisiones Arquitectónicas

Se documentaron dos decisiones clave:

* Uso de arquitectura cliente-servidor
* Implementación de base de datos relacional

Cada decisión incluye su contexto, solución adoptada y consecuencias.

---

### 🧱 Diseño del Sistema

Se definieron los principales componentes:

* Cliente (interfaz de usuario)
* Servidor (lógica de negocio)
* Base de datos (almacenamiento)

Además, se describe la interacción entre estos elementos mediante API.

---

### 📊 Diagramas UML

Se incluyen los siguientes diagramas:

| Tipo                  | Descripción                                 |
| --------------------- | ------------------------------------------- |
| Diagrama de clases    | Representa la estructura del sistema        |
| Diagrama de secuencia | Representa la interacción entre componentes |

---

### 📄 Plantillas

Se desarrollaron plantillas reutilizables para documentación:

| Plantilla      | Uso                                    |
| -------------- | -------------------------------------- |
| ADR            | Registro de decisiones arquitectónicas |
| Diseño Técnico | Documentación del sistema              |
| UML            | Documentación de diagramas             |

---

## 🎯 Propósito del Proyecto

El propósito de este trabajo es demostrar la importancia de la documentación en la fase de diseño de software, permitiendo organizar el desarrollo, justificar decisiones técnicas y facilitar el mantenimiento del sistema.

---

## 🛠️ Enfoque Utilizado

El desarrollo de este trabajo se basó en prácticas de ingeniería de software orientadas a:

* Documentación estructurada
* Modelado mediante UML
* Registro de decisiones arquitectónicas
* Organización técnica del sistema
